# Brainstorms — the WHAT

Exploratory requirements docs, one per topic, written by
[`/ce-brainstorm`](../../.claude/skills/ce-brainstorm/SKILL.md). This is where a
vague or ambitious idea gets thought through **before** it becomes a plan or an
issue — the front of the compound-engineering loop.

- File pattern: `YYYY-MM-DD-<topic>-requirements.md`.
- A brainstorm is **not** committed work. It does not become an issue
  automatically — it becomes one when you decide to act on it (the natural
  hand-off into `/ce-plan`).
- Output is a right-sized requirements doc (summary, problem frame,
  requirements with R-IDs, success criteria, scope boundaries) — see the skill.

Downstream: `/ce-plan` reads the matching `*-requirements.md` here and turns the
WHAT into a HOW under [`docs/plans/`](../plans/).
